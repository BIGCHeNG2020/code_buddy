# 技术文档

## 系统架构

本系统采用前后端分离架构：

- **前端**: React + Ant Design
- **后端**: FastAPI + LangChain + Chroma
- **向量数据库**: Chroma (本地持久化)

## 核心技术

### 1. 向量检索

使用Chroma向量数据库进行语义检索：
- 将文档分割成小块
- 使用Embedding模型转换为向量
- 存储到Chroma数据库
- 查询时计算相似度

### 2. LangChain框架

使用LangChain构建问答链：
- Retriever: 检索相关文档
- Prompt: 构建提示词
- LLM: 生成回答

### 3. Embedding模型

默认使用本地模型 `all-MiniLM-L6-v2`：
- 轻量级，无需API Key
- 支持中英文
- 向量维度：384

## API接口

### 提问接口
```
POST /api/ask
Body: {"question": "你的问题"}
Response: {
    "answer": "回答内容",
    "sources": [...],
    "has_answer": true
}
```

### 文档管理
```
GET /api/documents       # 获取文档列表
POST /api/documents/upload  # 上传文档
DELETE /api/documents/{filename}  # 删除文档
```

### 系统管理
```
GET /api/status     # 获取系统状态
POST /api/rebuild   # 重建知识库索引
```

## 配置说明

修改 `backend/config.py` 可以配置：

- LLM模型：支持OpenAI API或兼容接口
- Embedding模型：本地模型或OpenAI
- 文本分割参数：chunk_size, chunk_overlap
- 检索参数：top_k, score_threshold
